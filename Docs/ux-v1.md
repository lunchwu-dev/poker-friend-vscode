# Poker Friends V1.0 — UX 交互与视觉设计方案

> 本文档定义 V1 版本的交互流程、视觉规范和组件设计。
> 可交互 HTML 原型：[prototype/game-table.html](prototype/game-table.html)
> 配套文档：[V1 游戏设计](v1-game-design.md) | [V1 技术方案](v1-tech-design.md)

---

## 1. 设计原则

### 1.0 视觉调性

> **"深夜高端私局"** — 整体画面偏暗沉（墨夜蓝底色），通过绿色毛毡和金色高光
> 制造经典赌场的氛围。不追求卡通或花哨，而是克制、沉稳、有质感。
>
> **标志性设计点 (Signature)**：
> - 品牌标题使用 **Playfair Display** 衬线体（斜体粗体），赋予传统赌场优雅感
> - 全局 SVG 噪点纹理叠加（2-4% opacity），消除数字感，增加物理质感
> - 按钮顶部叠加半透明光泽层（gloss overlay），模拟玻璃按键效果
> - 牌桌毛毡使用多层 radial-gradient + 内阴影，模拟灯下凸起效果
> - 登录页浮动牌背装饰缓慢呼吸浮动，营造沉浸感

### 1.1 交互原则

| 原则 | 含义 |
|------|------|
| **信息层次清晰** | 最重要的信息（底牌、底池、操作按钮）最大最醒目 |
| **一目了然** | 牌桌状态 1 秒内可感知 — 谁在操作、底池多少、我的牌是什么 |
| **最少操作步骤** | 核心操作（跟注/弃牌）一键完成，不需要多余步骤 |
| **容错防误** | All-in 等不可逆操作需二次确认，弃牌需远离其他按钮 |
| **竖屏优先** | 单手可操作，拇指热区覆盖操作面板 |

## 2. 色彩规范

### 2.1 主色板

| 用途 | 色值 | 色名 | 说明 |
|------|------|------|------|
| **外层背景** | `#080E1A` | 墨夜蓝 | 最深层背景，配合 radial-gradient 制造氛围光晕 |
| **次级表面** | `#0F1A2E` | 深藏蓝 | 卡片/面板等浮层背景 |
| **牌桌中心** | `#0f7a4a` | 翡翠绿 | 毛毡中央最亮处 |
| **牌桌中层** | `#0D5E3A` | 深绿 | 毛毡径向渐变中间带 |
| **牌桌边缘** | `#094d2e` | 暗绿 | 毛毡外圈，形成内凹深度 |
| **牌桌边框** | `#1A7A4E` | 中绿 | 牌桌边缘高光线 |
| **金色主调** | `#D4A843` | 金色 | 品牌色：赢牌、高亮、CTA |
| **金色亮部** | `#E8C76A` | 浅金 | 渐变起始色 / 文字高光 |
| **金色暗部** | `#B8912E` | 深金 | 渐变终止色 / 阴影 |
| **操作面板** | `#1A2A3A` → `#152234` | 暗灰蓝渐变 | 底部浮层，180° 线性渐变 |

> **视觉氛围**：非牌桌区域背景不使用纯色填充，而是采用 `radial-gradient` 混合多层光晕（金色 4-6% 不透明度 + 绿色 8% 不透明度），配以全屏 SVG feTurbulence 噪点纹理叠加（opacity 约 2-4%），营造深邃且有质感的空间。

### 2.2 操作按钮色

所有按钮使用 **180° 线性渐变** + **顶部光泽层**（`rgba(255,255,255,0.1)` → `transparent`），配合 `box-shadow` 制造立体感。

| 按钮 | 渐变色 | 文字色 | 阴影 |
|------|--------|--------|------|
| **弃牌 Fold** | `#C53030` → `#9B2C2C` | `#FFFFFF` | `0 4px 12px rgba(197,48,48,0.3)` |
| **过牌 Check** | `#3182CE` → `#2B6CB0` | `#FFFFFF` | `0 4px 12px rgba(49,130,206,0.3)` |
| **跟注 Call** | `#3182CE` → `#2B6CB0` | `#FFFFFF` | `0 4px 12px rgba(49,130,206,0.3)` |
| **加注 Raise** | `#38A169` → `#276749` | `#FFFFFF` | `0 4px 12px rgba(56,161,105,0.3)` |
| **All-in** | `#E8C76A` → `#B8912E` | `#1A1A1A` | `0 4px 20px rgba(212,168,67,0.3)` |

## 3. 字体规范

### 3.1 字体栈定义

| Token | 字体 | 用途定位 |
|-------|------|----------|
| `--font-display` | **Playfair Display**, Georgia, serif | 品牌标题、胜利文案等展示性文字。衬线体赋予赌场高端感 |
| `--font-body` | **Outfit**, -apple-system, sans-serif | 通用 UI 文字。几何无衬线体，现代且可读性佳 |
| `--font-mono` | **JetBrains Mono**, SF Mono, monospace | 筹码数值、金额、房间号等需对齐的数字。等宽确保对位一致 |

> **加载方式**：通过 Google Fonts CDN 按需加载 `wght@400;500;600;700;800;900`。React Native 端在 App 启动时预加载字体文件。

## 4. 图标与资源

### 4.1 需要的图标

| 图标 | 用途 | 样式 |
|------|------|------|
| 返回箭头 ← | 牌桌/房间页返回 | 白色线性，24px |
| 星标 ☆/★ | 收藏当前手牌 | 线性/实心，24px |
| Dealer 标记 D | 庄家位置 | 绿色圆形白字，20px |
| 加号 + | 空座位（点击入座） | 虚线圆形，48px |
| 分享图标 | 房间分享 | 白色线性，24px |
| 复制图标 | 复制房间码 | 白色线性，20px |

### 4.2 扑克牌素材

牌面使用 **角标 (corner-index) + 中央花色** 布局，模拟真实扑克牌外观：
- **左上角标**：点数 + 花色符号，自然阅读方向
- **右下角标**：同左上，旋转 180°
- **中央**：大号花色符号作为视觉焦点
- **牌面材质**：`linear-gradient(165deg, #fff, #f4f0e8)` 仿真纸张质感
- **牌背**：`linear-gradient(145deg, #162848, #0e1e38)` + 内嵌虚线框 + 45° 斜线纹理

| 素材 | 规格 | 说明 |
|------|------|------|
| 牌面（正面） | 纯 CSS 渲染 | 纸张渐变 + 角标布局 + 中央大花色，圆角 8px |
| 牌背 | CSS 渐变+纹样 | 深蓝渐变 + 内框 + 对角线纹理，圆角 8px |
| 大尺寸底牌 | 68×96px | 屏幕底部展示自己的底牌，hover 时上浮 4px |
| 小尺寸公共牌 | 48×68px | 牌桌中央公共牌 |
| 迷你牌（他人） | 28×40px | 结算时显示其他玩家底牌 |
| 赢家展示牌 | 56×80px | 结算弹层赢家手牌 |

## 5. 布局系统

### 5.1 屏幕分区 (375×812pt, iPhone 尺寸基准)

```
┌─────────────────────────────────────┐ ← 安全区顶部
│  顶部信息栏 (44pt)                    │    返回 | 底池 | 盲注 | 收藏
├─────────────────────────────────────┤
│                                     │
│  牌桌区域 (约 420pt)                  │    椭圆牌桌 + 座位环绕
│  · 上方玩家 (2-3人)                   │
│  · 左侧玩家 (1-2人)                  │
│  · 右侧玩家 (1-2人)                  │
│  · 公共牌 (中央)                     │
│  · 底池显示 (公共牌上方)              │
│                                     │
├─────────────────────────────────────┤
│  底牌区域 (100pt)                    │    自己的 2 张底牌，大尺寸
├─────────────────────────────────────┤
│  操作面板 (80pt)                     │    弃牌 | 跟注 | 加注
├─────────────────────────────────────┤
│  阶段进度条 (32pt)                   │    开局→翻牌→转牌→河牌
└─────────────────────────────────────┘ ← 安全区底部
```

### 5.2 座位分布（最多 9 人）

自己始终在屏幕底部中央，其他玩家按如下位置分布：

```
          [P4]    [P5]    [P6]
                                        ← 牌桌上方
   [P3]                        [P7]     ← 牌桌两侧

   [P2]    ┌──────────────┐    [P8]     ← 牌桌中部
            │   公共牌      │
            │   底池        │
            └──────────────┘

                [P1/我]                  ← 固定底部
```

不同人数的座位使用情况：

| 人数 | 使用座位 |
|------|---------|
| 2 人 | P1(我) + P5(正对面) |
| 3 人 | P1 + P4 + P6 |
| 4 人 | P1 + P3 + P5 + P7 |
| 5 人 | P1 + P3 + P5 + P6 + P7 |
| 6 人 | P1 + P2 + P4 + P5 + P6 + P8 |
| 7 人 | P1 + P2 + P4 + P5 + P6 + P7 + P8 |
| 8 人 | P1 + P2 + P3 + P4 + P5 + P6 + P7 + P8 |
| 9 人 | 全部使用 |

### 5.3 玩家座位组件

```
        ┌──────┐ ┌──────┐
        │  底牌 │ │  底牌 │    ← 结算时显示（平时隐藏）
        └──────┘ └──────┘
          ┌──────────┐
          │  (头像)   │        ← 圆形 48px，弃牌时加半透明遮罩+"弃牌"
          └──────────┘
     ⓓ     昵称                ← Dealer标记（如果是庄家）
          ┌──────────┐
          │  12,500   │        ← 筹码数，深色圆角背景
          └──────────┘
             200               ← 当前轮下注额（如果有）
```

当前操作者增加：
- 座位外围**金色发光边框**
- **环形倒计时**进度条包裹头像

## 7. 动画规范

### 7.1 V1 必须实现的动画

| 动画 | 时长 | 缓动 | 描述 |
|------|------|------|------|
| **发底牌** | 300ms/张 | ease-out | 牌从牌桌中央飞向各玩家位置，先到牌背，自己的牌自动翻转 |
| **翻公共牌** | 200ms/张 | ease-in-out | 牌背水平翻转为正面，依次翻开（间隔 150ms） |
| **下注筹码** | 250ms | ease-out | 圆形筹码图标从玩家位置滑向底池区域 |
| **赢得底池** | 400ms | ease-in | 底池筹码滑向赢家位置 |
| **弃牌** | 200ms | ease-in | 手牌向牌桌中央缩小 + 淡出 |
| **操作倒计时** | 30s（线性） | linear | 环形进度条围绕当前操作者头像，颜色从绿→黄→红渐变 |
| **结算 YOU WIN** | 500ms | spring | 文字从缩放 0 弹出到正常大小，带金色光效 |
| **按钮按下** | 100ms | ease-out | 轻微缩放 0.95，释放回 1.0 |
| **加注面板** | 250ms | ease-out | 从屏幕底部向上滑入 |

### 7.2 动画优先级

| 优先级 | 动画 | 说明 |
|--------|------|------|
| P0（必须） | 操作倒计时、按钮反馈 | 不实现影响可用性 |
| P0（必须） | 翻公共牌、发底牌 | 核心游戏体验 |
| P1（应该） | 下注筹码、赢得底池 | 增强游戏反馈 |
| P1（应该） | 弃牌、结算动画 | 增强视觉效果 |
| P2（可以延后） | 加注面板滑入 | 可用 static 替代 |

## 8. 组件规范

### 8.1 扑克牌组件

**正面**：
- 白色圆角矩形（radius 8px）
- 左上角：点数 + 花色图标（小号）
- 中央：大号花色图标
- 右下角：点数 + 花色图标（旋转180°）
- 红心/方块为红色，黑桃/梅花为黑色

**背面**：
- 深蓝色圆角矩形
- 中央简单图案（菱形格或 Logo）

**尺寸变体**：

| 变体 | 尺寸 | 用途 |
|------|------|------|
| 大 | 70×100px | 自己的底牌 |
| 中 | 50×72px | 公共牌 |
| 小 | 36×52px | 结算时其他玩家底牌 |
| 迷你 | 24×34px | 玩家座位旁的牌背（发牌后） |

### 8.2 玩家座位组件

**状态变化**：

| 状态 | 视觉表现 |
|------|---------|
| 空位 | 虚线圆形 + 「+」号 |
| 已入座/等待中 | 头像 + 昵称 + 筹码 |
| 游戏中 | 头像 + 昵称 + 筹码 + 当前下注额 |
| 当前操作者 | 金色边框 + 倒计时环 |
| 已弃牌 | 头像半透明 + 「弃牌」标签覆盖 |
| All-in | 头像 + 「All-in」标签 + 金色边框 |
| 赢家 | 金色光效 + 赢得金额动画 |

### 8.3 操作按钮组件

- 圆角矩形，高度 48px
- 弃牌按钮宽度较窄（约 25%），与其他按钮间距更大
- 跟注/加注按钮均分剩余空间
- 按钮内显示操作名 + 金额（如「跟注 400」）
- 禁用状态：opacity 0.4，不可点击

### 8.4 底池显示组件

- 位于公共牌上方
- 深色半透明背景标签
- 「总底池」+ 金额，金色数字
- 有边池时在下方依次显示「边池1:xxx  边池2:xxx」

### 8.5 阶段进度条组件

- 位于屏幕最底部，高度 32px
- 四个节点：开局 → 翻牌 → 转牌 → 河牌
- 已过阶段：绿色实线，节点实心
- 当前阶段：绿色节点闪烁
- 未到阶段：灰色虚线，节点空心
- 右侧显示倒计时 00:XX

## 10. 响应式适配

### 10.1 屏幕适配策略

| 屏幕 | 适配方式 |
|------|---------|
| iPhone SE (375×667) | 基准尺寸，座位间距略压缩 |
| iPhone 15 (393×852) | 标准展示 |
| iPhone 15 Pro Max (430×932) | 间距增大，底牌区更宽敞 |
| Android 中端 (360×780) | 与 iPhone SE 类似处理 |

### 10.2 适配原则

- 牌桌区域使用百分比布局，自适应宽度
- 座位位置使用绝对定位 + 相对父容器百分比
- 底牌和操作面板固定贴底，间距使用 SafeArea
- 公共牌和底池在牌桌中心，不受屏幕大小影响
- 字体最小不低于 12px（可读性底线）

## 11. 无障碍设计

| 要求 | 实现 |
|------|------|
| 色彩对比度 | 文字与背景对比度 ≥ 4.5:1 |
| 触摸目标 | 所有可点击元素最小 44×44pt |
| 花色区分 | 除颜色外，花色图标形状清晰可辨（不依赖色觉） |
| 操作反馈 | 按钮有明显按下状态（缩放 + 色变） |
