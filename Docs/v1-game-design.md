## 2. 德州扑克规则（完整实现）

### 2.1 游戏流程

```
开始新一手牌
    │
    ▼
① 确定庄家（Dealer Button）位置
    │
    ▼
② 强制下盲注（小盲 + 大盲）
    │
    ▼
③ 发底牌（每人 2 张，仅自己可见）
    │
    ▼
④ Pre-flop 下注轮（从大盲左边开始）
    │
    ▼
⑤ 发翻牌（3 张公共牌）→ Flop 下注轮
    │
    ▼
⑥ 发转牌（第 4 张公共牌）→ Turn 下注轮
    │
    ▼
⑦ 发河牌（第 5 张公共牌）→ River 下注轮
    │
    ▼
⑧ 摊牌（Showdown）→ 比较牌力 → 分配底池
    │
    ▼
⑨ 庄家按钮顺时针移动 → 下一手牌
```

### 2.2 下注规则

#### 2.2.1 盲注机制
- **小盲（Small Blind）**：庄家左手边第一位，强制下注小盲金额
- **大盲（Big Blind）**：庄家左手边第二位，强制下注大盲金额（= 2 × 小盲）
- 每手牌结束后，庄家按钮顺时针移动一位

#### 2.2.2 2人桌特殊规则
- 庄家 = 小盲（庄家先下小盲）
- 非庄家 = 大盲
- Pre-flop：庄家（小盲）先行动
- Flop/Turn/River：非庄家先行动

#### 2.2.3 下注轮规则
- 每轮下注从**特定位置**开始，顺时针进行
  - Pre-flop：从大盲左手边开始（UTG 位）
  - Flop/Turn/River：从庄家左手边第一位仍在手的玩家开始
- 玩家可执行以下操作：

| 操作 | 条件 | 说明 |
|------|------|------|
| **弃牌（Fold）** | 任何时候 | 放弃手牌，退出当前这手牌 |
| **过牌（Check）** | 当前无人下注时 | 不下注，传递给下一位 |
| **跟注（Call）** | 有人下注时 | 跟上当前最高注额 |
| **加注（Raise）** | 无限注：加注额 ≥ 上一次加注的增量 | 提高下注金额 |
| **全押（All-in）** | 任何时候 | 押上全部剩余筹码 |

#### 2.2.4 加注规则（无限注）
- 最低加注额 = 上一次加注的增量（若无人加注过，最低加注额 = 大盲额）
- 无上限（No-Limit）
- 示例：大盲 100，玩家A加注到 300（增量200），玩家B最低需加注到 500（300 + 200）

#### 2.2.5 下注轮结束条件
- 所有未弃牌的玩家下注额相同（或已All-in）
- 仅剩一名玩家时直接获胜（其他人都弃牌）

### 2.3 牌型判定（10 种，从高到低排列）

| 排名 | 牌型 | 英文 | 判定规则 | 示例 |
|------|------|------|---------|------|
| 1 | 皇家同花顺 | Royal Flush | A-K-Q-J-10 且全部同花色 | A♠ K♠ Q♠ J♠ 10♠ |
| 2 | 同花顺 | Straight Flush | 五张连续且同花色 | 9♥ 8♥ 7♥ 6♥ 5♥ |
| 3 | 四条 | Four of a Kind | 四张相同点数 + 一张踢脚牌 | K♠ K♥ K♦ K♣ A♠ |
| 4 | 葫芦 | Full House | 三张相同 + 两张相同 | Q♠ Q♥ Q♦ 9♣ 9♠ |
| 5 | 同花 | Flush | 五张同花色（不连续） | A♦ J♦ 8♦ 6♦ 3♦ |
| 6 | 顺子 | Straight | 五张连续点数（不同花） | 10♠ 9♥ 8♦ 7♣ 6♠ |
| 7 | 三条 | Three of a Kind | 三张相同点数 | 7♠ 7♥ 7♦ K♣ 2♠ |
| 8 | 两对 | Two Pair | 两组对子 + 一张踢脚牌 | J♠ J♥ 4♦ 4♣ A♠ |
| 9 | 一对 | One Pair | 一组对子 + 三张踢脚牌 | 10♠ 10♥ K♦ 8♣ 3♠ |
| 10 | 高牌 | High Card | 无任何组合 | A♠ J♥ 9♦ 6♣ 3♠ |

#### 2.3.1 同牌型比较规则
1. **同牌型**时，比较组成牌型的关键牌点数（例：两个三条，比三张的点数）
2. 关键牌相同时，比较**踢脚牌（Kicker）**，从高到低逐张比较
3. 五张牌完全相同时，**平分底池**
4. **花色不影响牌力高低**

#### 2.3.2 特殊顺子规则
- A 可作为最高牌（A-K-Q-J-10）或最低牌（5-4-3-2-A）
- A-2-3-4-5 是最小的顺子（称为"轮子"）
- K-A-2-3-4 不算顺子（A 不能同时做高低）

### 2.4 底池计算

#### 2.4.1 主池（Main Pot）
- 所有玩家的下注汇入主池
- 正常情况下，赢家获得全部主池

#### 2.4.2 边池（Side Pot）
- 当玩家 All-in 筹码不足以匹配其他人的下注时，产生边池
- All-in 玩家只能赢取主池（及其有资格参与的边池）
- 计算规则：
  1. 最小 All-in 额 × 参与人数 = 主池
  2. 剩余金额在有资格玩家间形成边池
  3. 可能产生多个边池（多人不同金额 All-in）

**边池示例**：
```
玩家A All-in 500，玩家B All-in 1000，玩家C Call 1000
  → 主池 = 500 × 3 = 1,500（A/B/C 都有资格）
  → 边池1 = 500 × 2 = 1,000（仅 B/C 有资格）
```

#### 2.4.3 底池分配
1. 从最小的边池开始，逐个结算
2. 每个池中牌力最大的有资格玩家获胜
3. 平局时均分对应池（不可整除的筹码给靠近庄家左边的玩家）

### 2.5 庄家按钮与位置

- 游戏开始时随机决定首个庄家位置
- 每手牌结束，庄家按钮顺时针移动一位（跳过已离座玩家）
- 位置名称（满桌 9 人为例）：

```
         [BB]  [UTG]  [UTG+1]
    [SB]                      [MP]
  [Dealer]                  [MP+1]
         [CO]    [HJ]
```

## 3. 用户系统

### 3.1 注册/登录方式

| 方式 | 优先级 | 说明 |
|------|--------|------|
| 手机号 + 短信验证码 | P0（首选） | 国内用户主要登录方式 |
| Apple Sign In | P0 | iOS 上架必须支持 |
| 微信登录 | P1 | 国内用户高渗透率 |
| 游客登录 | P0 | 降低进入门槛，后续可绑定手机号 |

### 3.2 用户资料

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| 昵称 | 游客默认生成，可修改 | 是（自动生成） |
| 头像 | 默认系统头像，支持从预设中选择 | 是（默认分配） |
| 用户ID | 系统生成的唯一标识（6-8位数字） | 自动生成 |
| 手机号 | 用于登录和找回账号 | 否（游客可不填） |
| 筹码余额 | 初始赠送 10,000 金币 | 自动 |

### 3.3 游客登录流程

```
首次打开 App
    │
    ▼
自动生成游客账号（设备ID绑定）
    │
    ▼
分配随机昵称 + 默认头像 + 10,000 金币
    │
    ▼
直接进入主界面（可立即开始游戏）
    │
    ▼
稍后可在「设置」中绑定手机号/微信（防丢号）
```

### 3.4 筹码系统（V1 简化版）

| 项目 | 设定 |
|------|------|
| 初始筹码 | 注册赠送 10,000 金币 |
| 每日补给 | 每日首次登录赠送 2,000 金币 |
| 破产补助 | 筹码 < 500 时可领取 3,000 金币（每日 3 次） |
| 筹码上限 | 无上限 |
| 筹码用途 | 仅用于游戏内下注 |

> V1 不设商城/内购，筹码仅通过游戏和每日补给获取。重点是让玩家有足够筹码随时能玩。

## 4. 房间系统

### 4.1 创建房间

房主可配置以下参数：

| 参数 | 选项 | 默认值 |
|------|------|--------|
| 房间人数 | 2 / 3 / 4 / 5 / 6 / 7 / 8 / 9 人 | 6 人 |
| 小盲/大盲 | 50/100、100/200、200/400、500/1000 | 100/200 |
| 最低买入 | 大盲 × 20 ~ 大盲 × 50 | 大盲 × 40 |
| 最高买入 | 大盲 × 50 ~ 大盲 × 200 | 大盲 × 100 |
| 操作时间 | 15秒 / 30秒 / 60秒 | 30秒 |
| 房间密码 | 可选，4位数字 | 无 |

### 4.2 房间邀请方式

#### 4.2.1 房间码
- 创建房间后生成 **6 位数字房间码**（如 `385721`）
- 其他玩家在主界面输入房间码即可加入
- 房间码有效期 = 房间存续期间

#### 4.2.2 分享链接
- 生成格式：`https://pokerfriends.app/room/{roomCode}`
- 支持通过系统分享面板发送到：微信、QQ、短信、复制链接
- 接收方点击链接 → 已安装 App 则直接跳转进入房间 → 未安装则跳转下载页
- 链接包含房间码，App 通过深度链接（Deep Link）解析

#### 4.2.3 邀请流程

```
房主 创建房间
    │
    ├─► 显示房间码 385721
    │
    ├─► [复制房间码] 按钮 → 复制到剪贴板
    │
    └─► [分享链接] 按钮 → 打开系统分享面板
              │
              ├─► 微信/QQ/短信...
              │
              └─► 生成带房间码的链接

好友 收到邀请
    │
    ├─► 方式1: 在 App 内输入房间码 → 加入房间
    │
    └─► 方式2: 点击分享链接 → Deep Link 打开 App → 自动加入房间
```

### 4.3 房间状态与生命周期

```
创建房间（Waiting）
    │
    ▼
等待玩家加入（至少 2 人才可开始）
    │
    ▼
房主点击「开始游戏」→ 进入牌局（Playing）
    │
    ▼
牌局进行 → 一手牌结束 → 自动发下一手
    │
    ├─► 有人离开 → 剩余 ≥ 2 人继续，< 2 人暂停等待
    │
    ├─► 房主点击「暂停」→ 当前这手打完后暂停
    │
    └─► 所有人离开 → 房间销毁
```

### 4.4 座位管理

| 规则 | 说明 |
|------|------|
| 入座 | 加入房间后选择空座位，确认买入金额后入座 |
| 买入 | 筹码从账户扣除，进入桌面筹码 |
| 补筹码 | 在两手牌之间可以补充筹码（不超过最高买入） |
| 站起 | 随时可站起（当前这手打完后生效），桌面筹码退回账户 |
| 离开房间 | 站起后可离开房间 |

### 4.5 房主权限

| 权限 | 说明 |
|------|------|
| 开始游戏 | 至少 2 人入座后可开始 |
| 暂停游戏 | 当前手打完后暂停 |
| 踢出玩家 | 将某位玩家踢出房间（当前手结束后生效） |
| 关闭房间 | 当前手结束后关闭房间，所有玩家桌面筹码退回 |

### 4.6 房间列表

V1 不做公共房间列表。玩家只能通过房间码/链接加入房间，确保私人房间属性。

## 5. 战绩系统

### 5.1 个人战绩面板

```
┌───────────────────────────────────┐
│         📊 我的战绩               │
├───────────────────────────────────┤
│                                   │
│  总场次          156              │
│  胜场            52               │
│  胜率            33.3%            │
│                                   │
│  总盈亏          +23,400          │
│  最大单局盈利     +8,600          │
│  最佳牌型        同花顺 ♠        │
│                                   │
├───────────────────────────────────┤
│         📋 最近牌局               │
├───────────────────────────────────┤
│  03/07 20:35  6人桌  +2,400  胜 ✅│
│  03/07 20:12  4人桌  -1,200  负   │
│  03/07 19:48  6人桌  +800   胜 ✅ │
│  03/07 19:21  3人桌  -2,000  负   │
│  ...                              │
└───────────────────────────────────┘
```

### 5.2 战绩数据项

| 数据项 | 说明 | 计算方式 |
|--------|------|---------|
| 总场次 | 参与的手牌总数 | 每参与一手牌 +1（发底牌后算参与） |
| 胜场 | 赢得底池的手牌数 | 获得底池（含平分）+1 |
| 胜率 | 胜场 / 总场次 | 百分比，保留一位小数 |
| 总盈亏 | 所有牌局的累计盈亏 | Σ（每手牌赢得 - 每手牌投入） |
| 最大单局盈利 | 单手牌最大盈利金额 | 取历史最大值 |
| 最佳牌型 | 打出过的最高牌型 | 记录最高牌型首次出现 |

### 5.3 最近牌局列表

| 字段 | 说明 |
|------|------|
| 时间 | 手牌结束时间 |
| 桌型 | 几人桌 |
| 盈亏 | 该手牌盈亏金额（+/-） |
| 结果 | 胜/负/平 |

- 保存最近 **100 手**牌局记录
- 滚动查看，按时间倒序排列

### 5.4 战绩存储
- 每手牌结算时，服务端记录该手牌每位参与玩家的参与和盈亏数据
- 汇总数据（总场次、胜率等）实时更新到用户 Profile
- 最近 100 手详情存数据库，超出部分保留汇总数据

## 7. 网络通信

### 7.1 通信方案
- **协议**：Socket.IO（基于 WebSocket，自动降级到 Long Polling）
- **传输格式**：JSON
- **连接方式**：客户端启动后建立持久连接，房间内通过 Socket.IO Room 广播

### 7.2 核心事件定义

#### 客户端 → 服务端（请求）

| 事件名 | 载荷 | 说明 |
|--------|------|------|
| `room:create` | `{ maxPlayers, smallBlind, bigBlind, minBuyin, maxBuyin, actionTime, password? }` | 创建房间 |
| `room:join` | `{ roomCode, password? }` | 通过房间码加入房间 |
| `room:leave` | `{}` | 离开房间 |
| `seat:sit` | `{ seatIndex, buyinAmount }` | 选座入座 |
| `seat:stand` | `{}` | 站起 |
| `seat:rebuy` | `{ amount }` | 补充筹码 |
| `game:start` | `{}` | 房主开始游戏 |
| `game:action` | `{ action: 'fold'|'check'|'call'|'raise'|'allin', amount? }` | 玩家操作 |

#### 服务端 → 客户端（推送）

| 事件名 | 载荷 | 接收者 | 说明 |
|--------|------|--------|------|
| `room:created` | `{ roomCode, roomConfig }` | 房主 | 房间创建成功 |
| `room:state` | `{ players[], seats[], config }` | 全房间 | 房间状态快照 |
| `room:player_joined` | `{ player }` | 全房间 | 有人加入房间 |
| `room:player_left` | `{ playerId }` | 全房间 | 有人离开房间 |
| `game:hand_start` | `{ handNumber, dealerSeat, blinds }` | 全房间 | 新一手牌开始 |
| `game:deal_hole` | `{ cards: [Card, Card] }` | **仅当事人** | 发底牌（私密） |
| `game:deal_community` | `{ stage, cards: Card[] }` | 全房间 | 发公共牌 |
| `game:action_on` | `{ seatIndex, availableActions, timeout }` | 全房间 | 轮到某玩家操作 |
| `game:player_acted` | `{ seatIndex, action, amount, potTotal }` | 全房间 | 某玩家执行了操作 |
| `game:hand_result` | `{ winners[], pots[], playerHands[] }` | 全房间 | 手牌结算 |
| `game:error` | `{ code, message }` | 当事人 | 操作错误 |

### 7.3 断线重连

```
客户端检测到连接断开
    │
    ▼
自动尝试重连（指数退避：1s → 2s → 4s → 8s，最大间隔 10s）
    │
    ├─► 30秒内重连成功
    │       │
    │       ▼
    │   发送 reconnect 事件（携带 token + roomCode）
    │       │
    │       ▼
    │   服务端推送完整房间+游戏状态快照
    │       │
    │       ▼
    │   客户端恢复画面，继续游戏
    │
    └─► 30秒内未重连
            │
            ▼
        该玩家当前手自动弃牌
            │
            ▼
        连续 2 手未回来 → 自动站起，桌面筹码退回账户
```

### 7.4 心跳保活

| 参数 | 值 |
|------|-----|
| 心跳间隔 | 5 秒 |
| 超时判定 | 连续 2 次心跳无响应（10秒）→ 判定断线 |
| 重连窗口 | 30 秒 |

    user_id         UUID NOT NULL REFERENCES users(id),
    seat_index      SMALLINT NOT NULL,

-- 索引

### 8.5 安全基础（V1）

| 措施 | 实现 |
|------|------|
| 服务端权威 | 所有游戏逻辑在服务端运算，客户端仅渲染 |
| 底牌隔离 | 每位玩家仅收到自己的底牌数据，他人底牌不下发 |
| JWT 认证 | Socket.IO 连接时验证 JWT Token |
| Prisma ORM | 参数化查询，防 SQL 注入 |
| WSS 加密 | 生产环境强制 TLS 加密传输 |
| 输入验证 | 所有客户端消息经 class-validator 校验 |

## 9. 游戏引擎状态机

### 9.1 手牌状态流转

```
IDLE（等待开始）
  │
  ▼
DEALING（发底牌）
  │
  ▼
PRE_FLOP（翻前下注轮）
  │
  ├─► 仅剩1人 → SHOWDOWN
  ▼
FLOP（发翻牌 + 下注轮）
  │
  ├─► 仅剩1人 → SHOWDOWN
  ▼
TURN（发转牌 + 下注轮）
  │
  ├─► 仅剩1人 → SHOWDOWN
  ▼
RIVER（发河牌 + 下注轮）
  │
  ▼
SHOWDOWN（摊牌 + 结算）
  │
  ▼
SETTLE（分配底池 → 记录战绩）
  │
  ▼
IDLE（等待下一手）
```

### 9.2 操作超时处理

| 场景 | 处理 |
|------|------|
| 操作超时（30秒） | 自动执行：有权过牌 → 过牌；否则 → 弃牌 |
| 连续 2 手超时 | 自动站起该玩家 |
| 断线中的操作 | 同超时处理 |
