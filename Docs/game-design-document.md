# 移动端德州扑克好友对战游戏 — 游戏设计文档

---

## 1. 游戏概述

### 1.1 游戏名称
**Poker Friends（扑克好友）**

### 1.2 游戏类型
多人在线实时对战卡牌游戏（Texas Hold'em Poker）

### 1.3 目标平台
iOS / Android 移动端

### 1.4 核心定位
面向好友间休闲社交的德州扑克对战平台，强调**社交互动**与**公平竞技**，无真金交易，以虚拟筹码和段位排名为核心驱动。

### 1.5 目标用户
- 18-40岁，有社交需求的轻度/中度扑克玩家
- 希望与现实朋友远程对战的用户群体

## 2. 核心玩法设计

### 2.1 德州扑克规则（无限注德州 No-Limit Hold'em）

| 阶段 | 描述 |
|------|------|
| **发牌（Pre-flop）** | 每人发2张底牌（Hole Cards），第一轮下注 |
| **翻牌（Flop）** | 发3张公共牌，第二轮下注 |
| **转牌（Turn）** | 发第4张公共牌，第三轮下注 |
| **河牌（River）** | 发第5张公共牌，最后一轮下注 |
| **摊牌（Showdown）** | 剩余玩家比较牌力，最大牌型获胜 |

### 2.2 玩家操作

| 操作 | 说明 |
|------|------|
| **弃牌（Fold）** | 放弃当前手牌 |
| **跟注（Call）** | 跟上当前最高注额 |
| **加注（Raise）** | 提高下注金额 |
| **全押（All-in）** | 押上全部筹码 |
| **过牌（Check）** | 不加注直接传递（无人下注时可用） |

### 2.3 牌型大小（从高到低）

| 排名 | 牌型 | 说明 |
|------|------|------|
| 1 | 皇家同花顺（Royal Flush） | A-K-Q-J-10 同花 |
| 2 | 同花顺（Straight Flush） | 五张连续同花 |
| 3 | 四条（Four of a Kind） | 四张相同点数 |
| 4 | 葫芦（Full House） | 三条 + 一对 |
| 5 | 同花（Flush） | 五张同花色 |
| 6 | 顺子（Straight） | 五张连续点数 |
| 7 | 三条（Three of a Kind） | 三张相同点数 |
| 8 | 两对（Two Pair） | 两组对子 |
| 9 | 一对（One Pair） | 一组对子 |
| 10 | 高牌（High Card） | 无组合，比最大单牌 |

### 2.4 盲注结构

| 房间类型 | 小盲/大盲 | 最低买入 | 最高买入 |
|---------|----------|---------|---------|
| 新手场 | 50/100 | 2,000 | 10,000 |
| 进阶场 | 200/400 | 8,000 | 40,000 |
| 高手场 | 1,000/2,000 | 40,000 | 200,000 |
| 自定义房间 | 房主设定 | 房主设定 | 房主设定 |

## 3. 游戏模式

### 3.1 好友私人房（核心模式）
- **创建房间**：房主设定盲注、人数（2-9人）、买入范围、时间限制
- **邀请方式**：房间码 / 微信分享 / 好友列表直邀
- **房间密码**：可选设置，防止陌生人进入
- **旁观模式**：好友可旁观正在进行的牌局
- **房主权限**：踢人、暂停、调整盲注

### 3.2 快速匹配
- 系统自动匹配同级别玩家
- 2-6人桌，盲注根据玩家段位自动分配
- 支持中途离开（筹码保留30秒倒计时）

### 3.3 锦标赛模式（SNG / MTT）
- **SNG（Sit & Go）**：满人即开，6/9人桌，淘汰赛制
- **MTT（多桌锦标赛）**：定时开赛，支持16-64人
- **好友锦标赛**：房主创建私人锦标赛，设定奖池分配规则
- **盲注递增**：每5/10/15分钟自动升盲

### 3.4 单人练习
- AI陪练（初级/中级/高级难度）
- 牌局复盘与牌力分析

### 9.4 核心服务模块

| 模块 | 职责 |
|------|------|
| **Gateway** | 连接管理、协议解析、负载均衡 |
| **Lobby** | 用户登录、好友、俱乐部、排行榜 |
| **Room** | 房间创建/销毁、座位管理 |
| **Game Engine** | 牌局逻辑、发牌、比牌、底池计算 |
| **Match** | 快速匹配、锦标赛调度 |

- 服务端发牌与计算，客户端仅展示
- 底牌信息仅发送给对应玩家，不广播
- IP/设备检测：同桌玩家IP或设备相同时警告
- 行为分析：异常操作模式检测（如合谋打牌）
- 牌局日志全量记录，支持举报复查

## 10. 网络与同步

### 10.1 通信协议设计

```protobuf
// 玩家操作
message PlayerAction {
    int32  seat_id   = 1;  // 座位号
    int32  action    = 2;  // 1=fold 2=check 3=call 4=raise 5=allin
    int64  amount    = 3;  // 下注金额
}

// 游戏状态推送
message GameState {
    int32          stage         = 1;  // 阶段
    repeated Card  community     = 2;  // 公共牌
    int64          pot           = 3;  // 底池
    repeated Seat  seats         = 4;  // 各座位状态
    int32          current_seat  = 5;  // 当前操作座位
    int32          timeout       = 6;  // 剩余时间(秒)
}
```

### 10.2 断线重连
- 断线后30秒内自动重连，恢复牌局状态
- 超时未重连：自动弃牌（强制弃牌机制，无论是否为当前操作者）→ 若再次超时则自动站起
- 重连后服务端推送完整当前游戏快照
- 牌局中主动离场：同样触发强制弃牌，清理座位后广播房间状态
- 房主离场时自动将房主身份转移给房间内下一位玩家

### 10.3 延迟优化
- 就近接入节点（华东/华南/华北/海外）
- 心跳包间隔5秒，超时10秒判定断线
- 操作预测：客户端预渲染操作结果

| 月度精英赛 | 每月底 | 钻石以上段位参赛，限定奖励 |
| 好友对战周 | 不定期 | 好友对战额外经验/金币加成 |

- 锦标赛开赛提醒

## 14. 合规与安全

| 项目 | 措施 |
|------|------|
| **实名认证** | 接入国家实名认证系统，未成年人限制游戏时长 |

| **无赌博属性** | 虚拟筹码不可提现、不可交易，符合监管要求 |

## 16. 风险与应对

| 风险 | 应对策略 |
|------|---------|
| 合规风险（赌博嫌疑） | 严禁真金交易、筹码不可提现、公示随机算法 |
| 外挂/作弊 | 服务端权威、行为分析、举报机制 |
| 玩家流失 | 社交绑定（好友/俱乐部）、赛季制、日常活动 |
| 服务器稳定性 | 多节点部署、自动扩缩容、灰度发布 |
| 付费率低 | 丰富装饰内容、VIP特权、限时折扣活动 |
